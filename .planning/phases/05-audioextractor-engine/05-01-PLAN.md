**AudioExtractor.swift** — `public enum AudioExtractor` (caseless) with one static method:

```swift
public static func extract(url: URL) throws -> URL
```

Implementation steps inside `extract(url:)`:
1. Find ffmpeg: check ["/opt/homebrew/bin/ffmpeg", "/usr/local/bin/ffmpeg", "/usr/bin/ffmpeg"] using FileManager.default.fileExists. If none found, throw `AudioExtractorError.ffmpegNotFound` immediately (before any Process is created).
2. Validate input: if !FileManager.default.fileExists(atPath: url.path), throw `AudioExtractorError.inputNotFound(url)`.
3. Resolve collision-free output path: start with `url.deletingPathExtension().appendingPathExtension("mp3")`. While that path exists on disk, append `_1`, `_2`, etc. to the stem before the extension. Concretely: stem = url.deletingPathExtension().lastPathComponent; dir = url.deletingLastPathComponent(); candidate = dir.appendingPathComponent("\(stem).mp3"); if exists try "\(stem)_1.mp3", "\(stem)_2.mp3", etc. Use a loop with an integer counter starting at 1, capped at 999 to prevent infinite loops.
4. Run ffmpeg: create a Process with executableURL = URL(fileURLWithPath: ffmpegPath), arguments = ["-y", "-i", url.path, "-vn", "-acodec", "libmp3lame", "-b:a", "320k", outputURL.path]. Set process.standardOutput = FileHandle.nullDevice and process.standardError = FileHandle.nullDevice. Call process.run() and process.waitUntilExit().
5. Check exit code: if process.terminationStatus != 0, throw `AudioExtractorError.extractionFailed("exit code \(process.terminationStatus)")`.
6. Return outputURL.

Do NOT use @discardableResult — callers need the output URL. Do NOT silence errors from Process.run() — let them propagate as Swift errors.

private var tempDir: URL!

    guard let fixtureURL = Bundle.module.url(forResource: "GH010001", withExtension: "MP4") else {
        throw NSError(domain: "test", code: 1, userInfo: [NSLocalizedDescriptionKey: "Fixture not found"])
    }

    try FileManager.default.copyItem(at: fixtureURL, to: dest)
    return dest
}
