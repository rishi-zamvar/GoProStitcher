## How to Build and Test

```bash
# Build app
xcodebuild -project GoProStitcher.xcodeproj -scheme GoProStitcher -destination 'generic/platform=macOS' build

# Unit tests (inside SPM package)
swift test --package-path GoProStitcherKit

# Integration + UI tests via Xcode
xcodebuild -project GoProStitcher.xcodeproj -scheme GoProStitcher -destination 'platform=macOS' test

# Regenerate project after modifying project.yml
xcodegen generate
```

## Decisions Made

- Used xcodegen (found at /opt/homebrew/bin/xcodegen 2.44.1) rather than manual xcodeproj creation — project.yml is diff-friendly and regenerable
- GoProStitcherKitTests runs via `swift test`, not via xcodebuild scheme — SPM package tests are not Xcode targets
- Integration test target uses `bundle.unit-test` type with `TEST_HOST: ""` so it runs without app hosting (pure library test)
- swift-tools-version downgraded from generated 6.2 to 5.9 to match the plan's Swift 5.9+ requirement and macOS 13 target
