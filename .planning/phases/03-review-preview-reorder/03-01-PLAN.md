  <behavior>
    Three static async functions that read local MP4 metadata via AVFoundation:

    1. duration(url: URL) async -> TimeInterval?
       - Uses AVURLAsset and loads .duration
       - Returns nil if asset cannot be loaded or duration is indefinite/zero
       - input: URL to valid tiny MP4 fixture -> output: non-nil TimeInterval > 0
       - input: URL to non-existent file -> output: nil

    2. resolution(url: URL) async -> CGSize?
       - Uses AVURLAsset, loads first video track's naturalSize
       - Returns nil if no video track found
       - input: URL to valid tiny MP4 fixture -> output: non-nil CGSize (width > 0, height > 0)
       - input: URL to non-existent file -> output: nil

    3. thumbnail(url: URL) async -> NSImage?
       - Uses AVAssetImageGenerator, requests frame at time .zero
       - Returns nil if image generation fails
       - input: URL to valid tiny MP4 fixture -> output: non-nil NSImage
       - input: URL to non-existent file -> output: nil

    Key implementation constraints:
    - AVMetadataReader must be a caseless enum (matches GoProNameParser/FolderScanner pattern — pure static namespace, prevents instantiation)
    - Use AVURLAsset(url:) with try await asset.load(.duration) for duration (new async API, macOS 13+)
    - Use try await asset.loadTracks(withMediaType: .video) then track.load(.naturalSize) for resolution
    - Use AVAssetImageGenerator with appliesPreferredTrackTransform = true; call image(at: .zero) — use the throwing async variant if available, otherwise copyCGImage(at:actualTime:) inside a Task
    - Do NOT use deprecated synchronous AVAsset property accessors (they trigger deprecation warnings on macOS 14+)
    - import AVFoundation and AppKit (for NSImage); no UIKit
    - swift-tools-version 5.9: no Swift regex literals, no @preconcurrency unless needed; async/await is fine

    Test setup:
    - GoProFileFactory creates tiny valid MP4 fixtures (established in Phase 1)
    - Tests must call await to exercise the async API
    - For invalid-URL tests, use URL(fileURLWithPath: "/nonexistent/file.mp4")
    - setUp() pattern: use try? + XCTAssertNotNil (cannot throw on macOS XCTest)
    - Each test function must be async: func testDurationValidFile() async throws { ... }
